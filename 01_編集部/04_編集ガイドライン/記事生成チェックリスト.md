# 記事生成チェックリスト（再発防止策）

> 初記事のフィードバックから策定。記事生成の各フェーズで自動チェックする。

---

## Phase 1：執筆前ヒアリング（必須）

記事を書き始める前に、まーくんに以下を必ず確認する。

| 確認事項 | 理由 |
|---|---|
| **固有名詞・数値の事実確認** | 間取り（2LDK/3LDK/4LDK）、部屋数、金額等の数値は推測で書かない |
| **登場する部屋・場所の内訳** | 「何部屋あって、それぞれ何に使っているか」を正確に把握する |
| **はるさんの実際のリアクション** | 想像で書かず、実際のエピソードをヒアリングする |
| **NGにしたい表現** | まーくんが恥ずかしいと感じる表現がないか事前確認 |

### 根拠
- 初記事で「2LDK」と書いたが実際は「4LDK」だった → Geminiレビューで矛盾を指摘され3回修正が発生

## Phase 2：執筆時の自動チェック（Claude側で実施）

### 文章品質

| チェック項目 | 基準 | 対処 |
|---|---|---|
| **一文の長さ** | 60文字以内 | 超過した文は分割する |
| **語尾の連続** | 同じ語尾が3回連続しない | 体言止め・会話文・だ/である調を挟む |
| **短文の連打** | 体言止めが3つ以上連続しない | 少し長めの文を混ぜてリズムを作る |
| **数値の一貫性** | 記事内で間取り・数量が矛盾しない | タイトル・リード・本文・まとめで同じ数値か確認 |

### 表現NG

| NG表現 | 代替表現 | 理由 |
|---|---|---|
| 「好きです」（はるさんへの直接的愛情表現） | 「感謝です」「ありがたい」 | まーくんが恥ずかしい |
| 他のライフスタイルとの比較 | — | メディア方針 |
| DINKsの正当化・弁明トーン | 楽しさをただ共有する | メディア方針 |
| 「必ず」「絶対」等の断言 | 「〜の場合が多い」 | 読者配慮 |

### 論理チェック

| チェック項目 | 具体例 |
|---|---|
| **読者の疑問が前提と合っているか** | 4LDKなのに「どこで生活してるの？」は不成立。部屋数に合った疑問にする |
| **まとめが本文と矛盾しないか** | まとめの箇条書きだけ読んでも本文と一致するか |
| **記事冒頭の設定が途中で変わっていないか** | 冒頭で「4LDK」なのに途中で「2LDK」が出てきていないか |

## Phase 3：投稿時の自動処理（Claude側で実施）

| 処理 | 内容 | 根拠 |
|---|---|---|
| **スラッグをローマ字で設定** | ハイフン区切り（例：`4ldk-shumi-beya`） | 日本語スラッグで公開してしまった |
| **ふきだしをSWELLブロック形式で出力** | `<!-- wp:loos/balloon -->` 形式を使用 | 手動変換の手間を排除 |
| **アイキャッチ画像のアップロード＆設定** | メディアアップロード→ `featured_media` 設定 | 設定忘れ防止 |
| **カテゴリの自動設定** | 記事メタデータからカテゴリIDを指定 | — |
| **メタディスクリプション** | 120文字以内で設定 | SEO基準 |

### ふきだしブロックの正しい形式

```html
<!-- まーくん（左・グリーン） -->
<!-- wp:loos/balloon {"balloonIcon":"http://www.dinks-torisetsu.fun/wp-content/uploads/2026/03/まーくん.png","balloonName":"まーくん","balloonCol":"green"} -->
<p>セリフ</p>
<!-- /wp:loos/balloon -->

<!-- はるさん（右・レッド） -->
<!-- wp:loos/balloon {"balloonIcon":"http://www.dinks-torisetsu.fun/wp-content/uploads/2026/03/はるさん.png","balloonName":"はるさん","balloonCol":"red","balloonAlign":"right"} -->
<p>セリフ</p>
<!-- /wp:loos/balloon -->
```

## Phase 4：レビュー時のチェック

| チェック | 担当 | 基準 |
|---|---|---|
| `/editorial-check` | Claude | GATE 0〜3 通過 |
| Geminiレビュー | Gemini MCP or チャット | 不快・論理・読みやすさ・拡散の4観点 |
| まーくん監修 | まーくん | 体験の嘘がないか・はるさんのセリフに違和感がないか |

---

## Phase 5：公開時の最終確認

| 確認項目 | 方法 |
|---|---|
| スラッグがローマ字か | API応答のslugフィールドを確認 |
| アイキャッチが設定されているか | `featured_media` が0でないか確認 |
| カテゴリが正しいか | `categories` フィールドを確認 |
| ステータスがpublishか | `status` フィールドを確認 |

```
    ↓

    ↓
公開確認（Phase 5：最終チェック）
    ↓
公開
```
