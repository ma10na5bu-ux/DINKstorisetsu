# CLAUDE.md — デカ太郎メディア統合プロジェクト指示書

## プロジェクト概要

デカ太郎が運営する2つのメディアを一元管理するプロジェクト。

| | デカ太郎ラボ（ブログ） | note |
|---|---|---|
| **URL** | https://dekataro.com | https://note.com/fond_rhino4330 |
| **目的** | 収益化（SEO → アフィリエイト） | フォロワー獲得 |
| **記事の主語** | 読者（ハウツー・レビュー・比較） | 自分（体験・感情・内省） |
| **テーマ** | 柴犬・DINKs暮らし・お金・レビュー・おでかけ | キャリア・趣味・内省・柴犬 |
| **想定読者** | 検索流入（20〜40代夫婦・柴犬オーナー） | 共感層（職場の人も見る前提） |
| **文字数** | 2,000〜3,000字 | 1,000〜1,100字 |
| **公開方法** | wp-post.py（自動投稿） | 手動コピペ |
| **成功指標** | PV・収益 | スキ・フォロワー数 |
| **相互導線** | 現時点ではなし（将来検討） | 同左 |

### メディア棲み分けルール
- 同じテーマでも「主語」で切り分ける: noteは「自分がどう感じたか」、ブログは「読者が知りたい情報」
- noteには**具体的なお金の話を書かない**（職場の人が見る可能性があるため）
- noteに情報記事は書かない。ブログに個人的な内省・ポエムは書かない
- 詳細は `メディア運用方針.md` を参照

- Slack: #dinksトリセツ（Shibamediaワークスペース）

### ネタ帳
- ローカルファイルで管理: `01_編集部/ネタ帳.md`
- Notionは参照しない（2026-04-13移行済み）

### Gemini MCP連携

## 編集方針

### コンセプト
「30代夫婦と豆柴の暮らし」を発信するメディア。柴犬・DINKs暮らし・おでかけ・お金の4軸。

### トーンの3原則
1. ポジティブであること：暮らしの楽しさを伝える
2. クスッと笑えること：日常のおかしみ、読んでて口角が上がる文章
3. なるべく誰も傷つかないこと：他のライフスタイルを否定しない

### NGルール
- **他のライフスタイルとの比較をしない**（子育て世帯 vs DINKs 等）
- **他人や他のものを否定しない**
- DINKsを正当化・弁明するトーンにしない
- 「DINKsの方が勝ち組」的な表現は絶対NG
- 子どもがいる人を「大変そう」と見下す表現もNG
- **医療・健康に関する記事は対象外**（獣医師に相談を案内するにとどめる）

### 記事の判断基準
「この記事を読んだ人が、思わずスクショしてパートナーに送りたくなるか？」

### コンテンツタイプと比率
| タイプ | 目標比率 | 内容 |
|---|---|---|
| 笑い・あるある | 50%以上 | 柴犬あるある、夫婦あるある、笑える実体験 |
| 体験記事 | 30% | 実用情報を体験ストーリーで包んだ記事 |
| 知識記事 | 20%以下 | SEO入口記事。体験エピソードを冒頭に入れる |

### カテゴリ（6分類）
| カテゴリ | slug | 内容 |
|---|---|---|
| 柴犬 | shiba | 柴犬の基本・しつけ・あるある・柴犬との暮らし |
| おでかけ | outing | キャンプ・旅行・犬連れお出かけ・散歩 |
| 暮らし | life | 夫婦の日常・間取り・家事・テレワーク・住宅 |
| お金 | money | 家計管理・資産形成・投資 |
| グッズ・レビュー | goods-review | ドッグフード・家電・キャンプギア・ペット用品 |
| コラム | column | エッセイ・雑感・検証記事 |

- 書き手（デカ太郎）の一人称視点
- ヨメ氏（妻）・ハチ（豆柴）も登場
- 読者に語りかける文体（「〜ですよね」「〜ありませんか？」）
- ストーリーの中に情報を織り込む
- 読者層：20〜40代の夫婦・柴犬オーナー・柴犬に興味がある人

### タイトル

## 制作フロー

### Phase 1: ネタ選び＋ヒアリング（✅承認①）
1. プリフライトチェック（`.env` の WP認証キー存在確認のみ。Notionチェック不要）
2. `01_編集部/ネタ帳.md` を読み込んで💡アイデア欄から候補を抽出（ファイル1回読むだけ）
3. 候補＋ヒアリングテンプレートをセットで提示
4. デカ太郎が選択＋回答（1往復で完結）

### Phase 2: 執筆＋レビュー＋監修（✅承認②）
5. 既存記事1本を読んでトーン確認
6. 内部リンクのスラッグを事前確認
7. 執筆（Markdown形式）
8. 並列実行: editorial-check + Geminiレビュー + アイキャッチプロンプト生成
9. デカ太郎に監修依頼（体験の嘘チェック・面白いか）

### Phase 3: アイキャッチ画像（✅承認③）
10. プロンプトファイルパスを伝える
11. デカ太郎がGeminiで画像生成
12. trim-eyecatch.py でトリミング（自動）

### Phase 4: 公開（全自動）
13. 画像アップロード（curl -F multipart形式。--data-binaryはWAF 403）
14. wp-post.py で記事投稿（**必ず `--auto --publish --media <ID>` を付ける**）
    ```
    python3 02_技術部/スクリプト/wp-post.py "原稿ファイル.md" --auto --publish --media <画像ID>
    ```
    - `--auto`: YAMLフロントマターからtitle・slug・meta_descriptionを自動読み込み（必須）
    - `--media`: アイキャッチのmedia ID（`--eyecatch-id`は存在しない）
15. 公開後にtitle・slug・linkをAPIで確認。スラッグが`{id}-2`等になっていたら即修正
16. `01_編集部/ネタ帳.md` を更新（アイデア欄から削除→公開済み欄に追記）
17. git commit + push

### 派生記事
- 1体験→複数記事に切り出し、相互リンク
- トーン：デカ太郎一人称、日記より情報寄り。「友達に教えてあげる」距離感

## 品質チェック

### 二重チェック体制
- **editorial-check**: 構成・SEO・論理整合・AI特有エラー・医療情報混入チェック
- **Geminiレビュー**: 不快チェック・論理矛盾・読みやすさ・拡散チェック

### 文章品質ルール
- 一文60文字以内
- 同じ語尾が3回連続しない
- 体言止め3つ以上連続しない
- 数値の一貫性（タイトル・リード・本文・まとめで矛盾しない）

### 表現NG
- 「好きです」（ヨメ氏への直接的愛情表現）→「感謝です」「ありがたい」
- 「必ず」「絶対」等の断言 →「〜の場合が多い」
- 症状→具体的対処指示 → 獣医師に相談を案内

## 自律実行ルール

**基本原則：コスト（課金）が発生しない作業はすべて承認不要で自動実行。完了まで自律的に進行する。**

### 承認必須（これだけ）
- 外部の有料APIへの新規接続
- WordPressの設定変更（テーマ・プラグイン等）
- サイト全体に影響する一括変更
- 不可逆な本番環境の操作

### 共通制約
- 個人情報・APIキーをコードに直書きしない
- `.env`の内容をログに出力しない
- 記事は必ず /editorial-check を通してから投稿する

### 報告ルール
- 承認不要の作業 → 完了後に1行で事後報告
- エラー → 内容と対処を1行で報告、解決できたら続行
- 解決できないエラーのみ → 詳細を報告して指示を仰ぐ
- 作業完了時は git commit + push でGitHubを最新に保つ

## note 編集方針

### 概要
- ペンネーム: デカ太郎
- 方向性: 「AIで日々の課題を解決する37歳」の体験記メディア。働き方・ゴルフ100切り・キャンプ計画・筋トレ習慣化など、リアルな課題をAIで試行錯誤する過程を正直に発信
- 自動公開不可。原稿をローカルで作成し、手動でnoteにコピペ公開する運用

### noteコンテンツピラー（4本柱）
1. **キャリア・働き方** — マーケ→Ops転向、転職活動のリアル
2. **趣味** — ゴルフ、キャンプ、筋トレ
3. **内省・自分との向き合い方** — 言語化の苦手意識、飽き性、挫折
4. **柴犬** — エッセイ寄りの柴犬との暮らし（情報記事はブログ側）

※ お金はnoteのピラーから除外。具体的なお金の話はブログで書く

### note執筆ワークフロー

#### 1. ネタ出し（✅承認①: テーマ選択）
以下を並列実行し、ネタ案をS/A/B級で提示する。
- **市場調査**: noteのトレンド・お題企画・競合をWeb調査
- **過去データ分析**: 公開済み記事のスキ率・ビューから勝ちパターンを抽出
- **ネタ案作成**: 市場×過去データを掛け合わせてネタ案を提示
- `01_編集部/note/コンテンツ戦略.md` のピラー・カレンダーを参照
- **ブログ側のネタとの重複がないか確認する**
- デカ太郎がテーマを選択

#### 2. 素材収集
- デカ太郎への深掘り質問で体験のディテールを引き出す（ここが記事の質を決める）
- 関連する資料・データがあれば同時に確認する（職務経歴書、スクショ等）
- 過去記事との関連性や連載としての流れを確認する

#### 3. 構成・タイトル提案（✅承認②: 構成・タイトル）
- 構成案（パート分け・文字数配分）を提示
- タイトル案を3〜5案提示（過去タイトルのパターン分析を踏まえ、同じ型の連続を避ける）
- アイキャッチ文言案を提示（既存アイキャッチのトーンに合わせて短く）
- ハッシュタグ案も提示（3層構造: お題+共感+テーマ）

#### 4. 執筆
- 承認された構成・タイトルに基づいて下書き作成
- 下書きは `01_編集部/note/下書き/` に保存
- 本文は1,000〜1,100文字を目安にする
- 読者の共感・感情を動かすフック設計:
  - 冒頭: 1〜2文で「自分のことだ」と思わせる（共感フック）
  - 中盤: 正直な体験・感情で信頼を作る
  - 締め: 力まない前向きさで「スキを押したくなる」読後感

#### 5. 品質チェック（一括実行）
以下をまとめて実行し、修正まで完了してからデカ太郎に提示する。
- **事実確認**: 過去記事との矛盾がないか（`memory/article_contents.md` 参照）
- **noteルール照合**: 見出し・太字・画像・一文長さ・語尾3連続・段落・目次・区切り線
- **/copywriting チェック**: 共感フック・締めの余韻・トーンの一貫性
- **平易化**: 専門用語・業界用語をnote読者向けに変換
- チェック項目:
  - [ ] タイトルと内容にギャップがないか
  - [ ] 冒頭2文で「読み進めたい」と思えるか
  - [ ] 一文が長すぎないか（40〜60字目安）
  - [ ] 自虐・ユーモアが自然か（無理に入れていないか）
  - [ ] 最後の一文で気持ちよく終われるか
  - [ ] ハッシュタグは適切か（お題+共感+テーマの3層）

#### 6. 仕上げ
- 改行リズムの最終調整（スマホ表示を意識）
- 目次・区切り線の挿入
- アイキャッチ文言の最終確定
- 太字箇所・画像差し込み箇所の明示

#### 7. 公開後処理（公開のたびに必ず実行）
- ユーザーがnoteに手動公開後、URLを共有してもらう
- `01_編集部/note/記事一覧.md` にURL・公開日・スキ数・タグ・参加企画を追記
- 下書きを `01_編集部/note/下書き/` → `01_編集部/note/公開済み/` に移動
- `01_編集部/note/アイキャッチ/` の画像を `番号_タイトル.拡張子` にリネーム
- `memory/article_contents.md` に記事サマリを追記
- マガジンへの追加を案内（ピラーに合ったマガジンに）

### 伸びる記事の型（記事5の成功パターン）

#### 構造: ビフォーアフター × 正直さ
1. 冒頭で前回記事からの接続（続編感・リピーター誘導）
2. 過去の失敗・弱さを正直に告白（共感フック）
3. 「今回変えたこと」の具体的な工夫（読者にとっての学び）
4. 根底にある目的・動機の言語化（深み）
5. 小さな成長実感 + 過信しない等身大の締め
6. 「次回予告」で連載感を出す

#### 伸びる条件
- お題企画に参加している（フォロワー外への露出）
- 過去記事の続編になっている（既存読者の再訪）
- 「弱さ → 工夫 → 小さな変化」の流れがある
- 1,000〜1,100文字で読みやすい

### note記事フォーマット
- 見出し（h2）は「結論がわかる文」にする
- 見出し前には区切り線（`---`）を入れる
- 「かぎ括弧」で印象付けたいキーワードを囲む
- 箇条書きはノウハウ・比較・列挙で使う。ストーリー・笑い系の記事は散文で通す
- 他記事リンクはハイパーリンクではなくURL直打ち。文中ではなく段落の後に独立行で配置

### note可読性ルール（スマホ前提）
- 1文は40〜60文字程度（スマホで1〜2行）に収める
- 1段落は3〜4行以内
- 段落と段落の間には1行の空白を入れる
- ひらがなへの「開き」で認知負荷を下げる（例えば→たとえば、等→など、お勧め→おすすめ）

### note執筆時の提示事項
- 本文中の**太字にすべき箇所**を明示する
- **画像を差し込むべき箇所**と、その画像の内容イメージを提示する

### noteアイキャッチのルール

#### ピラー別テイスト
- **キャリア・内省系**: 黒背景 × 白文字（明朝体）× 左寄せ。Canvaで作成（1280 × 670px）
- **趣味・柴犬・暮らし系**: デカ太郎ラボと同じフラットベクターイラスト。Gemini Advancedで生成 → trim-eyecatch.pyでトリミング

#### 共通ルール
- タイトルとは別の「記事の中で一番刺さる一文」をアイキャッチのコンセプトにする
- タイトルと重複させない（一覧でアイキャッチ+タイトルが両方表示されるため）
- **文言は短く（4〜10字）**。一覧で目に入り、読める大きさを優先する
- テキスト型は文字数が多い場合、意味の切れ目で改行（2行まで）
- イラスト型は `01_編集部/ガイドライン/アイキャッチ画像プロンプト集.md` を参照

### note文体ガイド（デカ太郎のトーン）
- 一人称は基本的に省略、使う場合は「自分」「私」を文脈で使い分け
- 自虐・ユーモアを自然に交える（「見事なチキンっぷり」「ただのデカい置物」「挫折の回数だけならベテラン」）
- 体験ベースで正直に書く。カッコつけない。弱さや失敗も隠さない
- 読者に語りかけるようなカジュアルな文体。「〜なんですよね」「〜だなと」
